## 1. 게임 콘셉트

플레이어는 서퍼가 되어 **하나의 거대한 파도 면(wave face)** 위를 타고 해역을 가로지른다.
파도 위쪽(마루 부근)은 잔잔하고 안전하지만 점수가 낮고, 아래쪽(파도 골·거친 바다)으로 내려갈수록
속도감과 위험이 커지는 대신 점수 배율이 높아진다.
장애물은 오른쪽에서 다가오며, 각 장애물은 **고유한 예고 신호**를 먼저 보여준다.
신호를 읽고 라인을 바꾸거나 점프해 회피하며, **얼마나 위험하게, 얼마나 멀리 가느냐**가 점수를 가른다.

핵심 차별화 3축:
1. **위험-보상 라이딩** — 거친 바다 띠를 타면 ×1.5 + 보너스, 화면이 위험을 시각·청각으로 알림.
2. **읽는 회피** — 장애물 6종이 각기 다른 신호를 내고, 가짜 신호가 섞인다.
3. **바다를 건너는 여정** — 미니맵으로 출발항→목적지 항해 진척을 보여주고, 클리어 시 다음 해역으로 보드가 이동.

## 2. 핵심 이동 — 파도 면 라이딩

### 2.1 모델 (기존 `Lane` 폐기)

현재 `constants.js`의 이산 레인(`Lane.TOP/MID/BOT`, `LANE_Y`)을 폐기하고
**연속 수직 위치 `y`** 로 전환한다.

- 라이딩 가능 영역: `RIDE_TOP_Y` ~ `RIDE_BOTTOM_Y` (화면 세로 약 28% ~ 88% 구간).
- 입력: `↑` = 파도 위(안전)로 카빙, `↓` = 파도 골(위험)로 카빙.
- **관성 모델** (이산 점프가 아닌 연속):
  - `vy`(수직 속도)를 입력에 따라 가속, 무입력 시 감쇠.
  - 권장값: `STEER_ACCEL = 2600 px/s²`, `STEER_MAX = 520 px/s`, `STEER_DAMP = 0.86/frame`.
  - 보드는 `vy`에 비례해 **기울어짐(carve)** — 시각적으로 ±18° 틸트. (현재 파란 사각형 → 회전 가능한 보드 형태로)
- `↓`로 더 아래(거친 바다)를 탈수록 화면에 **물보라/속도선**이 강해져 "빠르게 내달리는" 체감.

### 2.2 점프

- `Space`: 파도 면에서 보드가 떠오름(기존 점프 물리 유지: `JUMP_VELOCITY -900`, `GRAVITY 2400`).
- 점프 중 `↑/↓`로 약간의 공중 제어 가능(기존 좌우 → 상하로 재해석).
- **착지 복귀(피드백 #8)**: 착지 시 수직 입력이 없으면 점프 직전 라이딩 라인으로 부드럽게 복귀(`y → preJumpY`, lerp 12/frame).
- **파도 출렁임(피드백 #8, 후순위)**: 라이딩 기준선 자체가 `sin` 으로 미세하게 출렁여 착지 위치에 변수를 준다.

### 2.3 점수 배율 (레인 배율 대체)

이산 `LANE_MULTIPLIER` 대신 **수직 위치 기반 연속 구간 배율**:

| 구간 | y 범위(라이딩 영역 비율) | 배율 | 의미 |
|---|---|---|---|
| 안전 (마루) | 상단 0% ~ 45% | ×1.0 | 잔잔 |
| 중간 | 45% ~ 70% | ×1.2 | — |
| **위험 (거친 바다 띠)** | 70% ~ 100% | **×1.5** | [3장](#3-위험-보상-거친-바다-띠) |

`score.js`의 `_applyMultipliers(base, lane)` → `_applyMultipliers(base, zoneMul)` 로 변경,
`zoneMul`은 플레이어 `y`로부터 계산한다.

## 3. 위험-보상: 거친 바다 띠 (차별화 #1 — 현재 0점 노출 해결)

> 현재 `score.js`의 `breakdown[DANGER_LANE]`는 **단 한 번도 가산되지 않아 결과 화면에 항상 0**으로 뜬다.
> 이번 개정의 1순위는 이 항목을 실제로 작동시키고, 화면이 위험을 알리게 하는 것.

### 3.1 띠 정의

- 라이딩 영역 하단 **70%~100%** 구간을 "위험 구간(거친 바다)"으로 지정.
- 비주얼: 더 어둡고 거친 물결, 흰 포말(whitecaps), 가장자리 비네트.

### 3.2 보상 로직 (`score.js`)

- 플레이어 `y`가 위험 구간 안에 있는 동안 **초당 DANGER 보너스 누적**(생존 점수와 동일한 누산 방식):
  - `ScoreEvent.DANGER_LANE` 에 `DANGER_PER_SEC = 30` × 콤보배율을 매초 가산.
- 위험 구간에서의 회피/퍼펙트 점프는 ×1.5 구간 배율이 자동 적용되어 추가로 더 받음.

### 3.3 연출 (`GameScene` / `HUDScene`)

- 위험 구간 진입 시:
  - 화면 가장자리 **빨강/주황 비네트 펄스**.
  - 화면 측면에 `"위험구간 ×1.5"` 텍스트(깜빡임).
  - **물보라 강화 + 속도선** + 미세한 화면 흔들림.
- 이탈 시 즉시 원복. (기획서 §3.3, §7.3 "화면 가장자리" 의도 충족)

## 4. 장애물과 신호 (차별화 #2 — 6종 전부 고유 비주얼)

### 4.1 6종 신호 비주얼 (`obstacle.js` `SignalInstance._redraw`)

> 현재 SPLASH·FIN만 그려지고 나머지는 노란 사각형 fallback. 6종 전부 구현한다.

| 장애물 | 신호 | 비주얼 (telegraph 진행 `p`=0→1) |
|---|---|---|
| 날치 FLYING_FISH | SPLASH | 물 튀는 확장 링 + 중심 물방울 (현재 ✅) |
| 상어 SHARK | FIN | 수면 위 등지느러미 삼각형 + 항적선 (현재 ✅, 항적선 추가) |
| 고래 WHALE | SHADOW | 수면 아래에서 **떠오르는 거대한 어두운 그림자 타원** (크기·농도 ↑) |
| 해파리 JELLYFISH | GLOW | **맥동하는 반투명 청록 발광 원** (alpha sin 맥동) |
| 문어 OCTOPUS | TENTACLE | 아래에서 **솟아오르는 구불구불한 다리 2~3개**(sin 곡선) |
| 번개 LIGHTNING | FLASH | 하늘 섬광 + **번개 윤곽선** 깜빡임 (세로 경고선) |

- 신호 위치 `y`: 이산 `LANE_Y[lane]` → **연속 목표 y**(아래 4.3) 기준.

### 4.2 가짜 신호 (유지 ✅)

- `isFake` 데이터·로직 유지. 가짜 신호는 회피 동작을 유발하되 실제 장애물은 안 옴.
- (후순위) 복합 신호·밤바다 시야 제한은 [9장 후순위](#9-구현-로드맵).

### 4.3 스폰 데이터 호환 (`spawnPatterns.js` / `stages.js`)

- 기존 200+ 이벤트는 `lane: T/M/B`를 사용 → **버리지 않고 수직 앵커로 재해석**:
  - `T → y비율 0.20`, `M → 0.55`, `B → 0.82`.
  - `B`(아래) 이벤트는 자연스럽게 위험 구간과 겹쳐 "위험한 라인 유도" 효과.
- 데이터 포맷에 `yFrac`(0~1) 옵션을 추가로 허용해 연속 배치도 가능하게 확장.
- 주석 오기 수정: `"20분/22분"` → `"20초/22초"` (피드백 ⚠️).

## 5. 온보딩 — 1스테이지 인게임 튜토리얼 (차별화 #3 — 통째 누락)

> 기획 §4.1: "별도 튜토리얼 없이 첫 10초 인게임 말풍선 가이드". 현재 코드엔 한 줄도 없음.

- `storage.tutorialCompleted` 플래그로 1회만 노출(이미 정의돼 있으나 미사용 → 사용).
- 1스테이지 시작 후 약 12초간 단계별 말풍선:
  1. (0s) `"↑ ↓ 로 파도를 타고 오르내려요"` — 첫 입력 시 다음 단계.
  2. (~3s) `"신호를 보고 미리 피하세요"` — 첫 신호 등장에 맞춰.
  3. (~6s) `"Space 로 점프해 뛰어넘기!"` — 첫 회피/점프 시.
  4. (~9s) `"아래 거친 바다는 위험하지만 점수 ×1.5!"` — 위험 구간 강조.
- 튜토리얼 동안 스폰 밀도를 낮추거나 첫 장애물 telegraph를 길게.
- 완료 시 `tutorialCompleted = true` 저장, 이후 스테이지/재플레이에선 미노출.

## 7. 미니맵 & HUD 재배치 (차별화 #5)

> 기획 §7.3 배치와 현재 코드가 어긋남. 정렬한다.

### 7.1 미니맵 (신규)

- 플레이 중 **상단 좌측 고정**: 출발항 → 목적지까지의 바다를 가로 막대로, 서퍼 마커가 진척에 따라 이동.
- 진척 = `stageTimer / stageDuration` (거리 환산). "바다를 건너는" 체감의 핵심 HUD.

### 7.2 HUD 레이아웃 (`HUDScene`)

| 위치 | 기존 | 개정 |
|---|---|---|
| 좌 | 점수 | **미니맵 + 거리/진척** |
| 중앙 | 스테이지명 | **남은 시간(또는 거리)** |
| 우 | 타이머 | **점수 · 콤보 · 현재 배율(×1.0/1.2/1.5)** |

- 현재 우측의 "레인 배율 3종" 표시는 **현재 구간 배율 단일 표시**로 대체(연속 모델 반영).

## 8. 비주얼 / 연출 (방향만, 후순위 다수)

- 파도 면: 다중 패럴랙스 유지(현재 ✅) + 라이딩 면을 명확히 보이는 굵은 파도 곡선 추가.
- 위험 구간 띠: 어두운 물결 + 포말 + 가장자리 비네트(3장).
- **클리어 → 세계지도 보드 이동 애니메이션**(후순위, 핵심 연출): 결과 후 보드가 다음 해역으로 미끄러져 이동.
- 해역별 색감 전환 / 사운드 / 플레이스홀더 → 스프라이트: 후순위.

## 9. 구현 로드맵

> 사용자 결정: **이번엔 기획서만**. 아래는 합의 후 진행할 작업 순서.

### A. 즉시 (차별화 핵심 — 작은 작업으로 "기획서대로 작동" 인상)
| # | 작업 | 주요 파일 |
|---|---|---|
| 1 | 파도 면 라이딩 이동 모델 전환 (Lane 폐기, 연속 y + 관성 + 카빙) | `constants.js`, `player.js`, `GameScene.js` |
| 2 | 위험 구간 띠 + DANGER 보너스 실가산 + 연출 | `score.js`, `GameScene.js`, `HUDScene.js` |
| 3 | 신호 6종 고유 비주얼 | `obstacle.js` |
| 4 | 1스테이지 인게임 튜토리얼 말풍선 | `GameScene.js`, `storage.js` |
| 5 | 실패 화면 피드백 보강 | `ResultScene.js`, `score.js`(콤보 끊김 추적) |
| 6 | 미니맵 + HUD 재배치 | `HUDScene.js` |

### B. 후순위 (폴리시·재미·확장)
- 황금 물고기 / 기상 이변 랜덤 이벤트(`onGoldenFish`/`WEATHER_EVENT_MUL` 실연결).
- 클리어→세계지도 보드 이동 애니메이션.
- 착지 복귀·파도 출렁임 디테일.
- 스테이지 고유 기믹(밤바다 시야 제한·추격파도·화면 흔들림 — `nightMode`/`screenShake` 플래그 실연결).
- 해역별 색감, 사운드(BGM/효과음), 플레이스홀더 → 스프라이트, 복합 신호.

### C. 영향받는 파일 맵 (참고)
- `constants.js` — `Lane`/`LANE_Y`/`LANE_MULTIPLIER` 제거, 라이딩 영역 상수 + 구간 배율 함수 추가.
- `player.js` — 연속 상하 라이딩 물리, 보드 카빙 틸트, 착지 복귀.
- `score.js` — 구간 배율, DANGER 누적, 콤보 끊김 추적.
- `obstacle.js` — 신호 6종, 연속 y 배치.
- `spawnPatterns.js`/`stages.js` — lane→yFrac 재해석, 주석 오기 수정.
- `GameScene.js` — 파도 면·위험 띠 렌더, 위험 연출, 튜토리얼.
- `HUDScene.js` — 미니맵, 레이아웃 재배치, 단일 배율 표시.
- `ResultScene.js` — 실패 피드백 보강.
