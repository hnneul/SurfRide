# SurfRide 개정 기획서 (v3)

> 목적: v2(연속 라이딩 + 위험 띠 + 신호 6종)에서 한 단계 더 나아가
> **"진짜 파도 위에서 균형 잡는" 서핑 손맛**을 핵심으로 삼는다.
> 3대 변경: ① 2축 균형(밸런스) + 와이프아웃 ② 밑에서 솟구치는 장애물(옆스크롤 폐기/축소)
> ③ 서퍼 높이 제한. 여기에 난이도 상향과 중독성 보강을 더한다.
>
> 본 문서는 현재 코드(Phaser 3 + Vite, 바닐라 JS)에 1:1 매핑되는 작업 지시서다.
> **v2를 대체**하며, v2 문서는 "현재 구현된 코드의 스냅샷"으로만 참고한다.

## 0. v2 → v3 한눈 요약

| 구분 | v2 (현재 코드) | v3 (개정) |
|---|---|---|
| 조작 축 | ↑↓ 높이 1축 + Space 점프 (←→ 미사용) | **↑↓ 높이 + ←→ 균형 + Space 점프 = 2축** |
| 균형 | 없음 (자유 라이딩) | **밸런스 게이지 + 와이프아웃**(한계 초과 시 즉시 종료) |
| 높이 이동 | 라이딩 영역 전체 자유 (`STEER_MAX 640`) | **좁은 가동 폭으로 제한** — 정밀 이동 + 점프 조합 강제 |
| 장애물 | 오른쪽 → 왼쪽 스크롤 | **신호 후 그 자리에서 솟구침**(변형 A/B 프로토타입 비교) |
| 난이도 | telegraph 단축·밀도로만 조절 | + 밸런스 드리프트·분출 속도·**판 내 강도 램프** |
| 중독성 훅 | 콤보·퍼펙트 점프 | + **Near-miss(아슬아슬 회피) 슬로우모션·보너스** |
| 스코프 | 6종 장애물 / 10해역 / 다수 미연결 기믹 | **4종 / 5~6해역으로 압축**, 죽은 코드 정리 |

구현 순서는 [8장 로드맵](#8-구현-로드맵).

## 1. 기술 스택 선택 (요청 항목)

### 1.1 현재 구성 분석
- **Phaser 3.88 + Vite 6 + 바닐라 ESM JS.** 에셋 없음(전부 절차적 도형), 물리 엔진 미사용(직접 운동학), 충돌은 AABB 수동.
- Phaser는 사실상 **씬 FSM · 입력 · Scale.FIT · 카메라 흔들림 · 게임 루프**만 쓰는 중. 렌더는 100% `add.graphics()`.

### 1.2 후보 비교

| 스택 | 장점 | 단점 | v3 적합성 |
|---|---|---|---|
| **Phaser 3 + Vite (유지)** | 씬/입력/스케일/카메라/트윈 즉시 사용. 마이그레이션 0. 새 메커니즘(균형·분출)은 전부 **운동학+상태머신**이라 추가 의존성 불필요 | 번들에 Phaser(~1.2MB) 포함, 실제론 일부만 사용 | **★ 최적** |
| Pixi.js + 직접 구현 | WebGL 렌더 빠름, 가벼운 코어 | 루프·씬·입력·스케일 직접 구현해야 함. 도형 위주라 렌더 이점 미미 | 과함 |
| 순수 Canvas2D + Vite | 의존성 0, 번들 최소 | 씬·입력·스케일·흔들림 전부 재구현 → 해커톤 시간 낭비 | 비추천 |

### 1.3 결정
- **Phaser 3 + Vite 유지.** 물리 엔진(Arcade/Matter)은 **계속 끈다** — 균형·분출은 강체 시뮬이 아니라 가벼운 운동학/타이머라 직접 구현이 더 정확하고 가볍다.
- **TypeScript 도입을 권장(선택).** 균형 × 분출 × 점수 상태가 얽히며 복잡도가 오르므로, `.js → .ts` 점진 전환으로 회귀 버그를 줄인다. 시간이 빠듯하면 v3 핵심 구현 후로 미뤄도 됨.
- telegraph→분출, 슬로우모션 등 시간 연출은 **Phaser Tween/Timeline**으로 처리(추가 라이브러리 불필요).

## 2. 핵심 조작 — 2축 서핑

```
↑ / ↓  : 파도 면에서 높이(라인) 이동 — 제한된 좁은 폭
← / →  : 균형 카운터-스티어 (보드 기울기 보정)
Space  : 점프 (공중에선 균형 드리프트 정지 = 호흡 구간)
```

`←→`가 v2에서 놀고 있었으므로 **균형 축으로 배정**해 새 기능이 조작을 늘리지 않는다.
실력 = 신호를 읽어 높이를 옮기는 판단 + 균형을 유지하는 손 + 점프 타이밍의 **세 박자**.

## 3. 균형 & 와이프아웃 (긴장형 — 사용자 확정)

### 3.1 모델 (`player.js`)
- 상태값 `tilt ∈ [-1, +1]` (보드 좌/우 기울기). 0이 안정.
- **드리프트**: 매 프레임 `tilt`가 한쪽으로 밀린다.
  `driftRate = BASE_DRIFT + DEPTH_K · depth(y) + waveWobble(t)`
  - `depth(y)`: 아래(거친 바다)일수록 ↑ → 위험-보상과 직접 연동(고배율 구간일수록 균형 어려움).
  - `waveWobble`: 라이딩 기준선의 `sin` 출렁임을 균형 외란으로 재사용(기획 §5.1 "파도 출렁임" 충족).
- **보정**: `←/→` 입력이 `tilt`를 0쪽으로 당김(`CORRECT_ACCEL`). 무입력 시 드리프트만 누적.
- **와이프아웃**: `|tilt| ≥ 1.0` → 넘어짐 연출(짧은 텀블) 후 **즉시 게임오버**. 충돌=즉시종료(기획 §1)와 동일 톤.
- **점프 중**: 드리프트 정지 → 공중이 "균형 리셋 호흡". 점프를 회피뿐 아니라 **균형 관리 수단**으로도 쓰게 해 전략성 부여.

### 3.2 권장 파라미터(초기값, 튜닝 전제)
- `BASE_DRIFT = 0.18 /s`, `DEPTH_K = 0.55`, `CORRECT_ACCEL = 1.6 /s²`(또는 속도형), 와이프아웃 텀블 `0.6s`.
- 해역이 올라갈수록 `BASE_DRIFT`·`DEPTH_K` 상승(난이도 축, [4장](#4-난이도-상향)).

### 3.3 연출/HUD (`GameScene` / `HUDScene`)
- 화면 하단 또는 보드 밑에 **수평선/밸런스 바**: `tilt`만큼 기울어 위험을 즉각 전달.
- `|tilt|` 0.7 초과 시 바 빨강 점멸 + 미세 흔들림.
- 보드 비주얼은 이미 `carve` 회전이 있으니 `tilt`를 합성(`setRotation(carve + tiltVisual)`).

## 4. 난이도 상향 — 서퍼 높이 제한 + 분출 + 드리프트

### 4.1 높이 제한 (`constants.js`, `player.js`)
- v2의 넓은 라이딩 폭을 **좁힌다**: `STEER_MAX 640 → ~400`, 가동 밴드 폭 축소(예: 라이딩 영역의 60%만 즉시 도달, 나머지는 가속 누적 필요).
- 의도: 위치를 "확 바꿔 도망"이 안 되게 → **작은 정밀 이동 + 점프**로만 회피. 분출과 결합 시 자연스럽게 어려워짐.

### 4.2 난이도 곡선 재설계
- telegraph 전반 `-100~150ms`, 분출 상승 속도 상향.
- **판 내 강도 램프**: 한 스테이지(60~80초) 동안 스크롤/드리프트/밀도를 선형 상승 → 매 판 후반 클라이맥스.
- 밸런스 드리프트(`BASE_DRIFT`)를 해역별 난이도의 새 축으로 사용.

## 5. 밑에서 솟구치는 장애물 (옆스크롤 폐기/축소)

> 사용자 결정: **두 변형을 프로토타입으로 만들어 비교 후 채택.**
> 플래그 `ERUPTION_MODE: 'A' | 'B'` (`constants.js`)로 토글해 동일 데이터로 양쪽 플레이 비교.

### 5.1 변형 A — 제자리 분출 (순수 반응형)
- 배경 파도/하늘은 계속 좌→우 패럴랙스(여정감 유지)지만 **장애물은 스크롤하지 않는다.**
- 흐름: 서퍼 근처 고정 구간(x ≈ 0.45~0.75 화면폭)의 특정 `y`에 **예고 신호** → `telegraphMs` 경과 → 그 자리에서 **아래→위 솟구침**(번개만 위→아래 내리꽂힘).
- 회피: 제한된 높이 내에서 그 `y`대를 벗어나거나, 낮은 분출은 `Space` 점프.
- 장점: 기획서 "신호 읽고 회피" 철학에 가장 충실, 리듬·반응 게임 손맛.
- 단점: 횡스크롤 러너의 "달려나간다" 체감 약화.

### 5.2 변형 B — 전방 분출 + 약(弱)스크롤 (절충)
- 장애물이 오른쪽에서 **수면 아래 그림자(신호) 상태로** 천천히 다가오다가, 서퍼 사정권 도달 시 **솟구침**.
- 횡스크롤 여정감 유지 + 분출 연출 + 기존 스크롤 코드 일부 재활용.
- 장점: "바다를 건넌다" 체감 유지. 단점: 순수 반응성은 A보다 약함.

### 5.3 공통 — 데이터 호환 (`spawnPatterns.js` / `obstacle.js`)
- 기존 200+ 이벤트의 `lane(T/M/B)`·`yFrac` = **분출 높이**, `t` = 신호 시각, `telegraphMs` = 분출까지 시간. **거의 그대로 재활용.**
- `ObstacleInstance.update`의 스크롤 이동을 **분출 트윈(rise/slam)** 으로 교체(A) 또는 분기(B).
- 충돌 판정(AABB)·`getJustPassed`는 "분출 완료 후 일정 시간 노출" 윈도우로 재해석.

## 6. 중독성 보강 (스코프는 줄이고 손맛은 늘림)

### 6.1 Near-miss — 추가할 1순위 훅
- 분출 장애물 히트박스 둘레에 **그레이즈 반경**. 닿지 않고 그 안을 통과하면:
  `NEAR_MISS` 보너스 + 짧은 슬로우모션(`time.timeScale` 0.4, 0.2s) + 효과음 + 콤보 가산.
- 점프 없이도 "아슬아슬하게 피했다"는 쾌감 → 닷지 게임 중독성의 핵심.

### 6.2 스코프 컷 (깊이 우선)
- **장애물 6종 → 4종**(날치·상어·고래·번개) 우선 구현. 해파리·문어는 후순위/컷.
- **해역 10 → 5~6**으로 압축, 각 해역에 뚜렷한 1기믹만.
- **죽은 코드 정리**: `score.js`의 `GOLDEN_FISH`·`WEATHER_EVENT`는 어디서도 호출 안 됨 → near-miss로 대체하거나 황금물고기 1개만 간단 연결.
- 가짜 신호(`isFake`)는 저비용·고재미 → **유지**.

## 7. 점수 · 연출 · HUD 영향

- `score.js`: `NEAR_MISS` 이벤트 추가, `DANGER_LANE`은 v2대로 유지(위험 구간 체류 보너스). 와이프아웃은 종료 사유로 결과 화면에 전달.
- HUD(`HUDScene`): v2 미니맵/콤보/배율에 **밸런스 바** 추가. 와이프아웃 임박 시 경고.
- 결과 화면(`ResultScene`): 실패 사유에 "와이프아웃(균형 상실)" / "분출 피격(놓친 신호)" 구분 표기.
- 튜토리얼(`GameScene` `TUTORIAL_STEPS`): "←→로 균형을 잡으세요" 단계 추가.

## 9. 프로토타입으로 결정할 것 (미해결)
- **분출 변형 A vs B** — 5장. 플레이테스트로 채택(이번 라운드 핵심 결정).
- 와이프아웃 페널티: 즉시 종료(현 제안) vs 짧은 스턴+시간손실. 우선 즉시 종료로 만들고 체감 보고 조정.
- 높이 가동 폭의 정확한 수치 — 분출 회피가 "타이트하지만 가능"한 지점 튜닝.
- 균형 드리프트 강도 곡선(해역별).
