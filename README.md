# 🌊 SurfRide (서프라이드)

> 파도를 넘어, 세계를 건너라 — 신호를 읽고 살아남는 제주 바다 서핑 액션 게임

브라우저에서 즐기는 서핑 회피 게임입니다. Phaser 3의 2D 게임 로직 위에 Three.js의 3D 바다를 입힌 하이브리드 렌더링으로, 제주 앞바다에서 태평양 신전까지 10개 스테이지를 헤쳐 나갑니다.

GitHub: https://github.com/hnneul/SurfRide.git

---

## 🏄 게임 소개

SurfRide는 수직으로 솟은 파도면을 타고 내려가며 바다에서 솟아오르는 장애물을 피하는 **"신호 읽기(read-the-signal)" 생존 액션** 게임입니다.

장애물은 무작정 날아오지 않습니다. 솟아오르기 전 물 위에 **텔레그래프(telegraph) 신호**가 먼저 나타나고, 플레이어는 그 신호를 보고 장애물의 종류와 위치를 읽어낸 뒤 **점프할지, 몸을 움직여 피할지** 순간적으로 판단해야 합니다. 장애물은 화면 한 지점에서 **제자리(in-place)로 솟구쳐(eruption)** 오르기 때문에, 핵심은 좌우 이동이 아니라 위아래 회피와 균형 유지입니다.

배경은 제주 바다를 모티프로 합니다. 현무암 절벽, 등대, 돌하르방 같은 제주의 풍경이 3D로 구현되어 있고, 파도와 물거품, 물보라가 실시간으로 출렁입니다. 한 판이 끝나면 마치 리포트처럼 펼쳐지는 **결과 화면**이 점수를 항목별로 분해해 보여주며, 다음 도전을 위한 피드백을 제공합니다.

## ✨ 핵심 특징

- **하이브리드 렌더링** — Phaser 3가 게임 로직·입력·점수·UI를 담당하고, Three.js가 3D 바다와 서퍼, 장애물 비주얼을 렌더링합니다. 2D 게임 상태를 매 프레임 3D 좌표로 변환해 동기화합니다.
- **신호 → 제자리 분출 모델** — 장애물마다 1:1로 대응하는 고유한 텔레그래프 신호(SPLASH·FIN·SHADOW·GLOW·TENTACLE·FLASH)가 먼저 나타난 뒤, RISE → ACTIVE → SINK 3단계로 제자리에서 솟구칩니다.
- **페이크 신호 & 조준 분출** — 일부 신호는 솟구치지 않는 가짜(bluff)이고, 일부는 플레이어의 현재 위치를 조준해 정확히 그 자리에서 분출합니다. 패턴이 100% 예측되지 않도록 설계되었습니다.
- **리포트형 채점** — 생존·회피·니어미스·퍼펙트 점프·위험 레인·황금 물고기·날씨 보너스를 항목별로 집계하고, 콤보 배수(최대 2.0배)와 위치 배수를 적용합니다.
- **월드맵 & 10개 스테이지** — 제주 앞바다부터 태평양 신전까지, 순차적으로 해금되는 10개 해역을 월드맵에서 선택해 도전합니다.
- **점진적 난이도** — 스테이지가 올라갈수록 새로운 장애물 종류가 해금되고 텔레그래프 시간이 짧아집니다.
- **진행 저장** — 클리어한 스테이지, 별, 최고 기록이 `localStorage`에 자동 저장됩니다.
- **반응형 DOM 오버레이 UI** — HUD·메인 메뉴·월드맵·일시정지 화면이 모두 HTML/CSS 오버레이로 구성되어 모바일부터 데스크톱까지 유연하게 대응합니다.

## 🎮 게임플레이

### 조작법

| 키 | 동작 |
| --- | --- |
| `↑` / `↓` | 위/아래로 가속 (파도면을 따라 수직 이동) |
| `←` / `→` | 균형 보정 (좌우 기울기 교정) |
| `Space` | 점프 (120ms 입력 버퍼, 착지 시 실행) |
| `P` / `ESC` | 일시정지 / 재개 |

> 플레이어는 **수직 이동만** 합니다. 좌우 화살표는 위치가 아니라 서퍼의 **균형(기울기)** 을 잡는 데 쓰입니다.

### 회피 방식

플레이어는 파도면의 3개 구역(상단 안전 / 중단 / 하단 위험)을 연속적인 Y 좌표로 오가며, 점프하거나 몸을 움직여 장애물을 피합니다.

- **점프 회피(jump)** — 신호를 읽고 `Space`로 뛰어넘습니다. (날치·상어·해파리·문어)
- **이동 회피(move)** — 점프가 통하지 않습니다(충돌 판정이 파도면 기준이라 뛰어도 소용없음). 신호를 보고 미리 위/아래로 비켜 분출하는 열에서 벗어나야 합니다. (고래·번개)

### 장애물

| 장애물 | 신호 | 회피 방식 | 비고 |
| --- | --- | --- | --- |
| 🐟 날치 (FLYING_FISH) | SPLASH (물보라 링 + 물방울) | 점프 | 텔레그래프가 빠르면 붉게 빛나는 `burst` 변종으로 등장 |
| 🦈 상어 (SHARK) | FIN (등지느러미 + 물살) | 점프 | — |
| 🐋 고래 (WHALE) | SHADOW (떠오르는 어두운 그림자) | 이동 전용 | 점프 불가 — 미리 위아래로 비켜야 함, 가장 큰 히트박스 |
| 🪼 해파리 (JELLYFISH) | GLOW (맥동하는 시안 빛 + 후광) | 점프 | — |
| 🐙 문어 (OCTOPUS) | TENTACLE (일렁이는 보라 촉수) | 점프 | — |
| ⚡ 번개 (LIGHTNING) | FLASH (노란 기둥 + 지그재그) | 이동 전용 | 위에서 내려옴, 점프 불가 |

신호는 스테이지에 따라 약 500~1200ms 동안 표시되며, 진행도 80% 지점에서 화면이 잠깐 흔들려 분출을 예고합니다(페이크 신호에는 이 흔들림 예고가 없습니다).

### ⭐ 황금 물고기 보너스

화면을 왼쪽에서 오른쪽으로 가로지르는 황금 물고기가 간헐적으로 나타납니다. 수집하면 **+500점**을 즉시 획득합니다.

### 점수 & 위험 레인 보너스

기본 점수는 다음과 같이 적립됩니다.

| 항목 | 점수 |
| --- | --- |
| 생존 | 초당 10점 |
| 회피 (dodge) | 100점 |
| 니어미스 (near-miss) | 150점 |
| 퍼펙트 점프 | 200점 |
| 위험 레인 체류 | 초당 30점 |
| 황금 물고기 | 500점 |
| 스테이지 클리어 | 1,000점 |

- **위치 배수** — 안전 구역(상단) ×1.0, 중단 ×1.2, **위험 구역(하단·거친 바다) ×1.5**. 위험을 감수할수록 점수가 커집니다.
- **콤보 배수** — 3콤보 이상 ×1.2, 6콤보 이상 ×1.5, 10콤보 이상 ×2.0.
- **날씨 보너스** — 기상 이변 구간에서는 획득 점수가 2배가 됩니다.

### 체력과 와이프아웃

- 체력은 하트 **3개**로 시작합니다. 충돌 시 1개씩 차감되며, 충돌 직후 1,100ms 동안 무적이 됩니다.
- 좌우 기울기(균형)가 한계치에 도달하면(`tilt ≥ 1.0`) **즉시 게임 오버(와이프아웃)** 됩니다. 기울기는 시간에 따라 누적되며 `←` / `→`로 교정합니다(공중에서는 자동 회복).

### 스테이지 클리어 조건

각 스테이지는 정해진 **제한 시간(45~80초)을 끝까지 살아남으면 클리어**됩니다. 클리어하면 다음 해역이 해금되고, 결과 화면에서 성과에 따라 별(0~3개)을 획득합니다.

## 🛠 기술 스택

| 영역 | 사용 기술 |
| --- | --- |
| 게임 엔진 (2D) | Phaser **3.88.2** |
| 3D 렌더링 | Three.js **0.184.0** |
| 빌드 도구 | Vite **6.0.0** |
| 언어 | Vanilla JavaScript (ES2022, ES 모듈) |
| 타입 검사 | JSDoc + `jsconfig.json` (lenient, `strict: false`) |
| 폰트 | Pretendard (한글 지원, CDN preload) |

### 하이브리드 렌더링 아키텍처

SurfRide는 **Phaser 2D 위에 Three.js 3D를 겹쳐 올리는** 구조입니다.

- **Phaser 2D** — 게임 로직, 입력, 충돌 판정, 점수 계산을 전담합니다. 3D 모드에서는 Phaser 캔버스를 `display: none`으로 숨깁니다.
- **Three.js 3D** — 시각 표현만 담당합니다. 3개의 파도 레이어로 구성된 절차적 바다, 서퍼, 장애물·신호 메시, 황금 물고기, 카메라 셰이크 등을 렌더링합니다.
- **브리지(bridge)** — 게임의 2D 좌표(고정 X 컬럼, 수직 Y)를 3D 월드 좌표로 변환합니다. 라이드 Y(높이)가 월드 Z(깊이)로 매핑됩니다.
- Three.js 캔버스는 `position: absolute` + `pointer-events: none`로 Phaser 컨테이너 위에 올라가며, 키보드 입력은 그대로 Phaser가 처리합니다.

게임 화면은 논리 해상도 **1920×1080**, FIT 스케일 모드로 종횡비를 유지하며 반응형으로 동작합니다. Three.js를 쓸 수 없는 환경을 위한 Phaser 2D 폴백 렌더링 경로(`OceanBackground`, `HUDScene`)도 갖추고 있습니다.

## 🚀 시작하기

### 사전 요구사항

- **Node.js** (ES 모듈 및 Vite 6 실행 가능한 버전)
- npm

### 설치

```bash
git clone https://github.com/hnneul/SurfRide.git
cd SurfRide
npm install
```

### npm 스크립트

| 명령어 | 설명 |
| --- | --- |
| `npm run dev` | Vite 개발 서버 실행 |
| `npm run build` | Vite 프로덕션 빌드 (`dist/` 생성) |
| `npm run preview` | 빌드된 `dist/`를 로컬에서 미리보기 |

```bash
npm run dev
```

개발 서버가 뜨면 브라우저에서 Vite 기본 주소인 **http://localhost:5173** 로 접속해 플레이할 수 있습니다.

> 장애물 스프라이트는 `scripts/generate-obstacle-assets.js`가 raw 픽셀 버퍼와 zlib 압축으로 PNG를 절차적으로 생성해 `img/` 디렉터리에 기록합니다. 별도의 `vite.config.js` 없이 Vite 기본 설정(HTML 엔트리 자동 감지)으로 동작합니다.

## 🧭 화면 흐름

```
BootScene (에셋 프리로드)
        ↓
메인 메뉴 (MainMenuScene)
   ├─ 시작하기 ──────────→ 스테이지 0
   ├─ 이어하기 ──────────→ 저장된 스테이지
   └─ 세계지도 보기 ──→ 월드맵 (WorldMapScene)
                              └─ 도전 ──→ 선택한 스테이지
        ↓
스테이지 플레이 (GameScene)
   ├─ P / ESC ──→ 일시정지 (PauseScene) ──→ 계속 / 메인으로
   ├─ 충돌·와이프아웃 ──→ 결과(실패) 
   └─ 제한 시간 생존 ──→ 결과(클리어)
        ↓
결과 화면 (ResultScene)
   ├─ 클리어 → 다음 해역 / 월드맵 / 다시하기
   └─ 실패  → 다시하기 / 월드맵
```

- **메인 메뉴**에서 새로 시작하거나 마지막 진행 지점에서 이어할 수 있고, 월드맵으로 이동할 수 있습니다.
- **월드맵**은 10개 스테이지 노드를 진행 상태(완료 ✓ / 진행 / 잠김)와 함께 보여주며, 선택한 스테이지로 도전합니다.
- **게임 씬**은 매 프레임 충돌·와이프아웃을 검사하고, 제한 시간을 넘기면 스테이지 클리어로 전환합니다.
- **결과 화면**은 클리어(별 연출)와 실패(원인 표시)를 다르게 보여주며, 점수 항목을 순차 애니메이션으로 카운트하고 최고 기록 비교·배지·힌트·격려 문구를 제공합니다. 애니메이션 중 `Space` / `Enter` 또는 클릭으로 건너뛸 수 있습니다.

> 스테이지 0(첫 플레이)에서는 튜토리얼이 활성화되어 단계별 힌트를 안내합니다.
